# 🚀 SmartYard-Server 1.0.0

[English](README.md) | [Русский](README.ru.md)

В этом релизе значительно расширены поддержка оборудования и мониторинг устройств Akuvox, BasIP, IS Sokol Plus,
Rubetek и Ufanet. Также добавлены массовое управление RFID-ключами, настраиваемое отображение камер и входов, новые
возможности мобильного API, поддержка Sesame DVR, ускоренное получение снимков и открытие дверей.

## ✨ Новые возможности

- Добавлена поддержка домофонов Akuvox S532, BasIP AA-07BD, AA-12FB и AA-14FB.
- Добавлена поддержка устройств Ufanet Secret Mini M2, Secret Solo и Secret Mole.
- Добавлена поддержка Rubetek RV-3437. Интеграция пока имеет статус бета-версии.
- Расширена поддержка реле RODOS для всей линейки устройств, вплоть до 16-канальных моделей.
- Добавлен мониторинг Zabbix для Ufanet Secret Mini, Secret Solo, Akuvox R20A и Akuvox S532.
- Добавлены новые реализации IS Sokol Plus для актуальных прошивок; предыдущие реализации оставлены как устаревшие
  модели.
- Добавлен тип медиасервера Sesame DVR.
- Добавлено массовое добавление RFID-ключей через веб-интерфейс и API.
- Добавлены настраиваемые через конфигурационный файл сервера режимы отображения входов в мобильном приложении.
- Добавлены режимы отображения групп камер и ручная сортировка камер в `allTree`.
- В мобильный inbox API добавлен метод отправки сообщений по номеру телефона абонента.
- Мобильный метод `getServices` теперь поддерживает отдельные списки услуг для домов из дополнительных полей.
- Расширены отслеживание событий в мобильном API и уведомления об открытиях из приложения, по коду доступа и по
  автомобильному номеру.
- Ответ мобильного FRS API теперь содержит `faceId` при отметке распознанного лица.
- Добавлены пользовательские группы лиц и опциональная кластеризация лиц с использованием FALPRS 1.1.0 или новее.
- В мобильный API добавлена настраиваемая поддержка Stories.
- Добавлены API-методы для импорта адресов и синхронизации договоров абонентов, состояния блокировки, телефонов и
  дополнительных полей из внешних биллинговых систем.
- Добавлены точки расширения формы камеры и загрузка локализации пользовательских подмодулей.
- В редакторы зон движения и распознавания камер добавлена возможность обновления снимка.
- Добавлен еженедельный запуск автоматического освобождения дискового пространства MongoDB GridFS.

## 🛠️ Улучшения

- Настройка режима работы LPRS теперь отображается только при выборе LPRS-сервера.
- Поля потоков камер теперь принимают URL с протоколами RTSP, HTTP и HTTPS.
- Унифицированы названия устройств в шаблонах Zabbix, используемых несколькими моделями домофонов.

## ⚡ Производительность

- Ускорены получение снимков и открытие дверей: исключены лишние проверки доступности устройства
  и запросы системной информации перед этими операциями.

## 🐛 Исправления

- Исправлены подключение и управление звонками из веб-интерфейса через Asterisk.
- Исправлена очистка значений `floor` и `openCode` через `modifyFlat()`.
- Исправлена нормализация RFID для Akuvox E12 и R20A, если код начинается с нуля.
- Исправлено количество выходов домофонов Sokol и Sokol Plus.
- Исправлена автонастройка камер Brovotech, iFLOW и Omny.
- Исправлен мониторинг SIP-регистрации домофонов IS Sokol Plus (rev.5) и Rubetek.
- Исправлены совместимость экспорта шаблонов с Zabbix 6.x и сопоставление ICMP-триггеров в Zabbix 7.x.
- Исправлено определение часового пояса адреса, если у дома не указан населенный пункт, но он указан у улицы.
- Исправлено отображение меню пользователя.

## ⚠️ Обновление

### Шаблоны Zabbix

Шаблоны мониторинга значительно изменились. После обновления SmartYard, если вы используете Zabbix, актуализируйте
конфигурацию бэкенда `monitoring` в соответствии
с [документацией](../../install/97.zabbix.md#configuration-smartyard-server) и импортируйте шаблоны повторно:

```bash
php /opt/rbt/server/cli.php --init-monitoring-config
```

### Обслуживание MongoDB GridFS

Установите еженедельное задание cron:

```bash
php /opt/rbt/server/cli.php --update-crontabs
```

Чтобы включить автоматическое освобождение дискового пространства GridFS, настройте файловый бэкенд:

```json
{
    "backends": {
        "files": {
            "backend": "mongo",
            "autocompact": "weekly"
        }
    }
}
```

Если вы используете ТТ - создайте рекомендуемый индекс метаданных GridFS для быстрой загрузки процессов,
фильтров, представлений и шаблонов печати:

```bash
php /opt/rbt/server/cli.php files --create-index=metadata.type,filename
```

### IS Sokol Plus (rev.5)

Рекомендуется [прошивка `2.5.0.15.30`](https://doc.is74.ru/books/umnyi-domofon-sokol-plius/page/cto-novogo-firmware).
Прошивка `2.5.0.14.13` остается совместимой, но обновление настоятельно рекомендуется. Существующие модели домофона
и камеры теперь помечены как устаревшие и переименованы в `IS SOKOL PLUS LEGACY (rev.5)` и `IS SOKOL LEGACY`.

После перехода на актуальные модели и запуска автонастройки проверьте корректность переноса настроек матрицы и КМС.
Устаревшая интеграция больше не поддерживается и будет удалена в одном из следующих мажорных релизов.

### Rubetek RV-3434

Рекомендуется [прошивка `2026.05`](https://support.rubetek.com/ru/access-control-system/firmwares/). Версии прошивки от
`2025.04` до `2026.05` продолжают поддерживаться как устаревшие. Более ранние версии больше не поддерживаются: перед
обновлением SmartYard обновите такие устройства до `2025.04`, а затем обновите устройство до прошивки `2026.05`.

Автонастройка домофонов Rubetek теперь использует UDP вместо TCP для транспорта SIP.

### Совместимость конфигурации

Параметры конфигурации `2faName` и `2faTitle` переименованы в `two_fa_name` и `two_fa_title`.
