# Обзор сервера

Серверная часть RBT реализована как набор в основном **ванильных PHP-скриптов** с **минимумом зависимостей**.
Проект намеренно избегает “тяжёлых” фреймворков: логика в коде явно выражена и проще для поддержки.

## Что означает “server” в этом репозитории

- **Точки входа** находятся в корне `server/` (например, `frontend.php`, `mobile.php`, `cli.php`, `asterisk.php` и т.д.).
- **Реализация API** разложена по папке `server/api/`.
- **Backends** — подключаемые реализации доменных сервисов (`server/backends/`).
- **Утилиты** и общие хелперы находятся в `server/utils/`.

## Основные характеристики (киты)

- **Ванильный PHP**: часто используется процедурный стиль; классы применяются точечно.
- **Минимум зависимостей**: Composer используется, но зависимости намеренно держатся небольшими.
