# PDOExt (server/utils/PDOExt.php)

`PDOExt` — наследник `PDO`, который добавляет:

- хранение исходного DSN и разбор DSN на `protocol` + параметры;
- удобные методы для частых операций: `insert()`, `modify()`, `modifyEx()`, `get()`, `queryEx()`;
- единый стиль обработки ошибок (через `setLastError()` + `error_log()`), с опцией подавления логирования (`silent`);
- для SQLite — регистрацию пользовательских функций `mb_strtoupper` и `mb_levenshtein`.

## Конструктор

**Сигнатура**: `__construct($_dsn, $username = null, $password = null, $options = null)`

Поведение:

- сохраняет DSN в приватное поле `$dsn`;
- вызывает `parent::__construct(...)`;
- выставляет `PDO::ATTR_ERRMODE` в `PDO::ERRMODE_EXCEPTION`;
- определяет протокол через `parseDsn()["protocol"]` и при `sqlite` регистрирует функции:
  - `mb_strtoupper` (1 аргумент),
  - `mb_levenshtein` (2 аргента).

Важно:

- регистрация функций делается всегда при `sqlite`, поэтому предполагается, что в окружении существуют callable `mb_strtoupper` и `mb_levenshtein`.

## parseDsn()

**Назначение**: разбирает строку DSN на протокол и список параметров.

**Возвращает**:

```php
[
  "protocol" => string,
  "params" => array,
]
```

Правила:

- DSN обязан содержать `:` (иначе процесс завершается через `die(...)`).
- Префикс (протокол) должен соответствовать `/^[a-z\d]+$/i` (иначе `die(...)`).
- Часть после протокола разбивается по `;`.
  - Если внутри DSN встречается `=`, элементы парсятся как `key=value`.
  - Иначе `params` становится списком из одного элемента (строка целиком после префикса).

Замечание по реализации:

- проверка на `=` делается через `strpos($dsnWithoutPrefix, '=')`, то есть **по всей строке**, а не по конкретному элементу — это означает, что при смешанных форматах поведение будет единым для всех элементов.

## trimParams($map)

Тримит пробелы у всех значений:

- если `$map` — list (`array_is_list($map)`), тримит каждый элемент;
- если ассоцициативный массив — тримит каждое значение;
- `null` значения оставляет как есть.

Используется в `insert()`/`modify()`/`modifyEx()` для предсказуемой передачи параметров в `execute()`.

## insert($query, $params = [], $options = [])

**Назначение**: выполнить INSERT (или иной запрос, где нужен `lastInsertId()`).

Поведение:

- `prepare($query)` → `execute(trimParams($params))`
- при успехе пытается вернуть `$this->lastInsertId()`
  - если `lastInsertId()` недоступен/бросает исключение — возвращает `-1`
- при `execute()` = false — возвращает `false`
- при ошибках:
  - если нет опции `"silent"`: `setLastError(...)`, логирует исключение и сам SQL
  - возвращает `false`

Возвращаемые значения:

- `string`/`int` (как вернёт `lastInsertId()`), либо `-1`, либо `false`.

## modify($query, $params = [], $options = [])

**Назначение**: выполнить модифицирующий запрос (UPDATE/DELETE/INSERT без необходимости `lastInsertId()`).

Поведение:

- `prepare($query)` → `execute(trimParams($params))`
- при успехе возвращает `$sth->rowCount()`
- при `execute()` = false — возвращает `false`
- обработка ошибок аналогична `insert()` (включая `"silent"`).

Возвращаемые значения:

- `int` (кол-во затронутых строк) либо `false`.

## modifyEx($query, $map, $params, $options = [])

**Назначение**: пакетно применить модификацию по списку полей (частичный update по карте соответствий).

Параметры:

- `$query`: строка-шаблон для `sprintf($query, $db, $db)`.
  - ожидается **2 плейсхолдера** под одно и то же имя колонки `$db` (например, для `SET col=:col` и/или для `WHERE`).
- `$map`: массив вида `dbColumn => paramName`
- `$params`: массив входных параметров (по именам `paramName`)

Логика:

- проходит по `$map`
- если в `$params` существует ключ `$paramName`, готовит запрос `sprintf($query, $db, $db)`
- выполняет с параметрами `[$db => $params[$paramName]]`
- суммирует `$sth->rowCount()` в `$mod`
- возвращает `$mod` (или `false` при ошибке)

Опция `"silent"` работает так же, как в `insert()`/`modify()`.

## queryEx($query)

**Назначение**: выполнить запрос без параметров и получить все строки.

Поведение:

- `prepare($query)` → `execute()`
- при успехе: `fetchAll(PDO::FETCH_ASSOC)`
- при неуспехе: возвращает пустой массив `[]`

Ошибки не перехватываются — исключение может выйти наружу (так как ERRMODE = EXCEPTION).

## get($query, $params = [], $map = [], $options = [])

**Назначение**: универсальный SELECT с опциональным маппингом результата и “схлопыванием” ответа.

Ветки выполнения:

- если `$params` не пустой:
  - `prepare($query)` → `execute($params)` (обратите внимание: тут **не используется** `trimParams()`)
  - `fetchAll(PDO::FETCH_ASSOC)`
- иначе:
  - вызывает `queryEx($query)`

Маппинг (`$map`):

- если `$map` задан, ожидается структура `sourceField => targetField`
- на каждую строку собирается новый массив `[$targetField] = $row[$sourceField]`

Опции (`$options`):

- `"singlify"`: если итоговый набор содержит ровно 1 строку — вернуть эту строку, иначе `false`
- `"fieldlify"`: если итоговый набор содержит ровно 1 строку — вернуть значение первого поля этой строки, иначе `false`
- `"silent"`: не логировать ошибки/SQL (для `PDOException`)

Возвращаемые значения:

- `array` (список строк или одна строка в режиме `"singlify"`)
- скаляр в режиме `"fieldlify"`
- `false` при несоответствии ожиданиям (`singlify`/`fieldlify`) или при ошибках.

## Внешние зависимости / ожидания окружения

- `setLastError(...)` должен существовать (используется в обработчиках исключений).
- Для `sqlite` должны быть доступны callable `mb_strtoupper` и `mb_levenshtein` (регистрируются как SQLite-функции).
